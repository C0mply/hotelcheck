# Project Management Approach

This project will follow a hybrid Scrum/Kanban approach.

Scrum principles will be used for planning work in short iterations around clear development goals, while Kanban will help manage the flow of tasks in a flexible way. This approach is suitable for a course project because the product can evolve during implementation, but still needs clear milestones and priorities.

Responsibilities will be divided between the human developer and the AI assistant.

I will be responsible for product direction, architecture, final technical decisions, code review, and validation of important changes. The AI assistant will support implementation by generating boilerplate code, suggesting refactorings, helping with documentation, drafting tests, and proposing small incremental improvements. All AI-generated changes should be reviewed before being accepted into the project.
